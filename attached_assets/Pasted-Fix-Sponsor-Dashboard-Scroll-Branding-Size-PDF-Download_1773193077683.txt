Part 1 — Sponsor Dashboard Fixes
1. Fix Sponsor Dashboard Scroll Behavior
Problem

On the Sponsor Dashboard, when a user scrolls down and then tries to scroll back up, they cannot return to the top of the page properly.

This suggests there is likely a layout or overflow issue such as:

a nested scroll container

overflow: hidden

overflow-y: auto on the wrong wrapper

a fixed-height content area trapping scroll

body scroll being disabled while an inner container scrolls

Required behavior

The Sponsor Dashboard should scroll normally like a standard page.

Users must be able to:

scroll down through all dashboard content

scroll back up to the top of the page

do this smoothly on desktop and laptop browsers

Required implementation steps

Check the Sponsor Dashboard layout and its parent containers for:

overflow: hidden

fixed heights like height: 100vh

nested scroll wrappers

content wrappers that trap scroll

Preferred fix

Use normal page scroll on the main document whenever possible.

If a custom content container is used, ensure:

only one vertical scroll container exists

it allows both downward and upward scrolling

the header / nav layout does not block scrolling

Acceptance criteria

user can scroll down through sponsor dashboard

user can scroll back to the top without getting stuck

no content area traps the scroll unexpectedly

2. Make Sponsor and Event Logos 150% Bigger
Problem

The sponsor and event logos on the Sponsor Dashboard are too small and do not have enough visual presence.

Required behavior

Increase the size of:

sponsor logo

event logo

to approximately 150% of their current displayed size.

Do this in a balanced way so the header still feels clean and not crowded.

Layout requirements

When increasing the logo size:

maintain aspect ratio

do not stretch logos

keep spacing clean

ensure the header still aligns well on desktop

preserve responsive behavior on smaller screens

Recommended implementation

Increase the rendered logo container size, not just the raw image.

Examples to review:

logo wrapper width/height

image max-width / max-height

flex alignment

spacing between logo and surrounding text

Acceptance criteria

sponsor logo is noticeably larger

event logo is noticeably larger

logos still look sharp and properly aligned

header remains visually balanced

3. Fix “Download PDF Report” on Sponsor Dashboard
Problem

The Download PDF Report action on the Sponsor Dashboard does not currently work.

Required behavior

When an authorized sponsor user clicks Download PDF Report, the system should:

generate or retrieve the sponsor performance PDF

return a valid downloadable PDF

download successfully in the browser

This must work for the authorized sponsor user.

Required checks

Investigate whether the issue is caused by:

broken route

missing auth handling

incorrect sponsor_id / event_id

bad file generation

failed response headers

server error during PDF build

front-end click handler not calling the correct endpoint

Required implementation steps

Validate end-to-end:

button click triggers correct action

request reaches correct sponsor report endpoint

server generates or fetches PDF successfully

response is returned with proper PDF headers

browser downloads file correctly

PDF download requirements

Set correct response headers such as:

Content-Type: application/pdf

Content-Disposition: attachment; filename="Sponsor_Report.pdf"

Use a meaningful filename when possible, such as:

[SPONSOR]_[EVENT]_Sponsor_Report.pdf

Example:

Winnow_FRC2026_Sponsor_Report.pdf
Permission requirement

Keep the export/download permission rule already defined:

only the primary Owner sponsor user can download/export data

Editors and Viewers should still be blocked.

Acceptance criteria

authorized sponsor owner can click Download PDF Report

valid PDF downloads successfully

unauthorized sponsor users cannot download it

no broken or empty file is returned
