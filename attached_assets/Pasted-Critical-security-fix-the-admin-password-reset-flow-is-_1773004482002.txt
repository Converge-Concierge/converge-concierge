Critical security fix: the admin password reset flow is publicly exposed and must be restricted immediately.

Problem:
The reset-password page is showing a reset token publicly in the browser UI. This must not be publicly accessible in production.

Required fixes:

1. REMOVE PUBLIC RESET TOKEN DISPLAY IN PRODUCTION
- Do not show reset tokens in the browser UI in production
- Do not show “Your reset token” or any dev preview token publicly
- This may only exist in development, behind an environment check

2. RESTRICT PASSWORD RESET FLOW
- Public users may request a reset only by entering email
- The UI should only show a generic message:
  “If an account exists for that email, reset instructions have been sent.”
- Never expose whether a specific admin email exists

3. ADMIN-ONLY RECOVERY PATH
Because email delivery is not yet reliable, create a secure fallback recovery path for admin use only.

Acceptable options:
- admin-only console utility/script to generate a one-time reset token
- protected server-side admin recovery endpoint disabled from public UI
- secure manual recovery process only accessible by authenticated system admin/dev

This fallback must NOT be visible on public pages.

4. PRODUCTION EMAIL RESET
If email sending is configured:
- send reset link/token by email only
- verify delivery logs
- verify SMTP/provider settings
- verify from-address/domain configuration

5. RESET TOKEN SECURITY
- one-time use only
- expires in 1 hour or less
- never rendered in public production UI
- never returned from the API to the browser in production

6. ROUTE PROTECTION
- review reset-related routes/pages
- ensure dev reset preview is disabled in production
- ensure no public page exposes raw reset tokens

7. IMMEDIATE GOAL
- allow Dan to reset the admin password safely
- remove public exposure of the reset token flow
- preserve a secure admin recovery method for future lockout cases

Success criteria:
- admin password can be reset safely
- no reset token is shown publicly in production
- public forgot-password flow only shows a generic success message
- a secure non-public fallback recovery path exists
