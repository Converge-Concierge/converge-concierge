Make the Welcome Page and Subsequent Cards Closely Match the Event Scheduling Page Layout
Use the TLS2026 event page as the visual/layout reference for the attendee onboarding experience

I would like the attendee welcome page and the subsequent onboarding cards to closely match the layout and visual structure of the main event scheduling page:

https://concierge.convergeevents.com/event/TLS2026

Right now the onboarding flow feels like a separate mini-app.
Instead, it should feel like a guided extension of the event scheduling page.

This is primarily a layout and design consistency task, not a change to the onboarding logic itself.

1. Primary Goal

Refine the attendee onboarding flow so that:

the welcome page visually matches the event scheduling page layout

subsequent cards also feel like they belong inside that same event page system

attendees feel like they are moving through a guided version of the event scheduling page, not switching into a different interface

2. Use the TLS2026 Event Scheduling Page as the Design Reference

The event scheduling page should be treated as the design/layout reference.

Specifically align the onboarding flow with the event page in terms of:

overall page width

hero/header treatment

event logo placement

event title sizing

event metadata row

section spacing

sponsor/session card proportions

typography hierarchy

button styling

white card / light background structure

Do not create a separate narrow wizard aesthetic if the goal is consistency with the event page.

3. Keep the Existing Event Page Hero Structure

The top of the welcome page should closely resemble the event scheduling page header.

Required structure

At the top of the page, use the same general hero pattern as the event page:

event logo on the left

title / event text on the right

badge or label row above title if appropriate

date/location/website metadata row below title

This should feel visually like the attendee has arrived on the event page, but now with a guided next-step experience layered into it.

The welcome page should still communicate that registration is complete and this is the next step.

Next Step label

The attendee should feel like they are progressing through sections of the event page, not stepping through a modal wizard.
