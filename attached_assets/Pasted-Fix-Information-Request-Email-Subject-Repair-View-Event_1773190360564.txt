Fix Information Request Email Subject, Repair “View Event Schedule” Button, and Add Admin Email Template Management

Implement the following email-related improvements in Concierge.

We need to correct the Information Request attendee confirmation email, fix the broken CTA link, and add a proper admin email template management function so all templates can be viewed and edited inside Concierge.

1. Update the Information Request Confirmation Email Subject

The attendee confirmation email for an information request currently uses the subject:

Your information request has been sent

We need this to be more specific and useful.

Required new subject format

Change the subject to:

Your information request has been sent to [Sponsor Name]
Example
Your information request has been sent to eGain
Required behavior

Wherever the attendee confirmation email for Information Requests is generated:

pass the sponsor name into the subject builder

use the sponsor display name from the sponsor record

do not hardcode the sponsor name

if sponsor name is missing, fall back to:

Your information request has been sent

But the normal path should always use the sponsor name.

2. Fix the “View Event Schedule” Button in the Email

The View Event Schedule button in the attendee confirmation email is currently not working.

We need to fix the CTA so it routes to the correct attendee-facing event scheduling page.

Required behavior

The button should link to the attendee-facing scheduling page for the relevant event.

Use the correct live route in Concierge for the event schedule page.

Example target pattern

Use something like:

/events/[event-code]/schedule

or whatever your actual attendee-facing scheduling route is.

The button should resolve dynamically based on the event tied to the Information Request.

Required implementation steps
2A. Determine the correct attendee-facing schedule URL

Identify the real public attendee-facing schedule route in the app.

The link must not point to:

admin routes

sponsor dashboard routes

placeholder routes

undefined URLs

It must point to the actual event sponsor discovery / meeting scheduling page.

2B. Pass event context into the email template

When generating the information request confirmation email, ensure the template has access to:

event id

event code or slug

event name

correct public schedule URL

2C. Generate CTA URL centrally

Add or use a helper function such as:

buildEventScheduleUrl(event)

This function should generate the correct public scheduling URL.

Use this helper inside the email template.

Do not hardcode event links in template HTML.

2D. If attendee prefill links already exist

If attendee-facing event pages support email lookup or secure token access, use that when generating the schedule URL.

For example, if your system already supports:

email query prefill

secure attendee token

event-specific personalized access

then include the appropriate parameter in the CTA link.

If that is not ready yet, just link to the correct public event scheduling page for now.

Required fallback behavior

If a valid event schedule URL cannot be generated, do not render a broken button.

Fallback options:

hide the button

or render a plain text message without CTA

Do not allow dead links in production email.

edit HTML body

save updates
