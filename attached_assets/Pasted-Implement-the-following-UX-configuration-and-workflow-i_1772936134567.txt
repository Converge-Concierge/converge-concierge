1. EVENTS — FIX MEETING CONFIGURATION + LOCATIONS
==================================================

Problem:
In Edit Event, Meeting Configuration and Meeting Locations are not working properly.

Required fixes:
- Make Meeting Configuration fully editable in Create/Edit Event
- Make Meeting Locations fully editable
- Persist changes correctly
- Display existing values correctly when editing an event

Each event must support:
- event-level meeting locations
- event-level meeting time blocks

NEW REQUIREMENT:
Not every location is available for every time block.

So each meeting time block must support selecting which locations are available for that block.

Implement this structure for each time block:
- Date
- Start Time
- End Time
- Available Locations (multi-select from the event’s defined meeting locations)

Behavior:
- Admin defines the master location list for the event
- For each time block, admin selects which of those locations are active in that block
- Public scheduling should only offer locations that are valid for the selected time block
- Admin scheduling should follow the same rule

Do not redesign the event modal.
Keep the current modal structure, but make these controls fully functional.
