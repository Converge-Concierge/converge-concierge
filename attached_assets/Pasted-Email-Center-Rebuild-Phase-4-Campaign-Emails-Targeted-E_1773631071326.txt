Email Center Rebuild — Phase 4
Campaign Emails (Targeted Event Communications)

We are now moving to Phase 4 of the Email Center rebuild.

This phase introduces Campaign Emails, allowing Converge to send targeted communications to event participants based on filters such as meeting activity, category, or sponsor relationships.

Campaigns are manual or scheduled operational communications, not automated system emails.

This is important.

Campaigns are used by Converge admins to:

• invite attendees to schedule meetings
• remind attendees to complete onboarding
• notify sponsors of new opportunities
• announce agenda updates
• communicate event updates

Campaigns should not replace automations. Automations are handled in Phase 3.

Campaigns are admin-controlled messaging tools.

Phase 4 Goal

Add a Campaigns system to Email Center that allows admins to:

• create targeted email campaigns
• filter audiences based on event data
• schedule or send campaigns
• track campaign activity in Email Activity

This will transform Concierge into a powerful operational communications tool for events.

5. Audience Preview

Before sending the campaign, admins must be able to preview the audience.

Add:

Preview Audience

This should display:

number of recipients

list preview (first ~20)

email addresses

name

company

Admins must confirm before sending.

This helps prevent mistakes.

6. Template Selection

Campaigns must use existing templates.

The campaign builder should include a template selector.

Display:

template display name

category

subject preview

Admins should also be able to preview the template before sending.

7. Send Options

Campaigns should support two send modes.

Immediately send campaign.

Schedule Send

Allow scheduling for a specific date/time.

Fields:

send date

send time

timezone (optional if system default exists)

Scheduled campaigns should appear in the Campaign list as:

Scheduled

9. Campaign Activity Integration

All campaign emails must appear in Email Activity.

Add source attribution:

Source: Campaign

Also include the campaign name.

Example:

Source: Campaign – FRC2026 Meeting Invitation

This ensures all communications remain traceable.

10. Campaign Metrics

Campaigns should track basic metrics.

Minimum metrics:

Emails Sent

Failures

Optional (future phases):

Opens

Clicks

For Phase 4, do not block launch on advanced analytics.

11. Campaign Safety Controls

Campaigns must include safety protections.

Recipient Validation

Before sending:

confirm recipient count

confirm event context

Admins must click:

Confirm Send

before final sending.

Sponsor Data Protection

Campaign system must not allow sponsor reports or sensitive sponsor data to be sent outside approved sponsor contacts.

Campaigns should only send:

operational communications

event updates

scheduling invitations

Sensitive sponsor reporting must remain restricted.

12. Campaign Email Limits (Safety)

Add basic safeguards to prevent accidental spam.

Example rule:

Max one campaign email per attendee per day

If this becomes restrictive later, it can be adjusted.

For now, prioritize safety.

13. Campaign Editing Rules

16. UI Design Goals

Campaigns should feel simple and operational.

Admins should be able to create a campaign in under 1 minute.

Avoid overly complex marketing automation features.

Concierge is an event operations platform, not a marketing platform.
