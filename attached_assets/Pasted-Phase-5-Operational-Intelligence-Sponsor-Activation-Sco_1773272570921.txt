Phase 5 — Operational Intelligence, Sponsor Activation Score, and Advanced Deliverables Monitoring

Phase 5 enhances the Concierge platform with operational visibility and sponsor success metrics.

The system should now help administrators:

monitor sponsor activation

identify sponsorship risks early

measure deliverables progress

surface operational problems automatically

demonstrate sponsor value more clearly

This phase does not introduce new core objects, but instead builds analytics and monitoring layers on top of the existing system.

Phase 5 Goals

Add operational intelligence so the system can answer questions like:

Which sponsors are not engaging with Concierge?

Which sponsors have incomplete deliverables?

Which sponsors have not scheduled meetings?

Which sponsors have not logged into their dashboard?

Which sponsors are fully activated?

This will help Converge protect sponsorship value and drive meeting activity.

1. Create Sponsor Activation Score

Create a Sponsor Activation Score that measures how engaged a sponsor is with the event.

The score should be calculated per:

sponsor

event

Store the score as a calculated metric, not a manually edited field.

1A. Activation Score Components

Score components should include:

Metric	Weight
Dashboard login	10
Agreement Deliverables completion	25
Meeting requests received	20
Meetings scheduled	20
Information requests received	10
Sponsor inputs completed	10
Profile completeness	5

Total possible score = 100.

1B. Score Calculation

Calculate the activation score dynamically or periodically.

Example formula concept:

activation_score =
login_score +
deliverables_score +
meeting_score +
info_request_score +
profile_score

Where each component contributes its weighted value.

1C. Activation Status Labels

Display score categories:

Score	Label
80–100	Fully Activated
60–79	Active
40–59	At Risk
0–39	Inactive

These labels should appear in admin views.

3. Deliverables Completion Metrics

Create a completion percentage based on:

completed_deliverables / total_deliverables

Exclude deliverables that are:

not sponsor visible

marked not applicable

Show this metric in:

sponsor agreement detail header

admin dashboard alerts

4. Admin Dashboard Deliverables Monitoring

Enhance the Admin Dashboard → Needs Attention section.

Add operational alerts.

4A. New Alert Types

Add alerts such as:

Sponsors with incomplete deliverables
Sponsors missing Certificate of Insurance
Sponsors with low activation score
Sponsors without meetings scheduled
Sponsors who have never logged in
Sponsors with outstanding sponsor inputs

4B. Alert Behavior

Each alert should:

display a count

link to a filtered list view

be event-specific if an event tab is selected

Example alert:

3 sponsors have incomplete deliverables

Link to:

Agreement Deliverables → Sponsor Agreements → Filtered

18. Acceptance Criteria

Phase 5 is complete when:

19. Success Standard

After Phase 5, Concierge should function as a true sponsor success monitoring system.

Admins should be able to quickly identify:

inactive sponsors

sponsors not scheduling meetings

sponsors not completing deliverables

sponsors not engaging with Concierge

This helps ensure every sponsor gets value from the event.
