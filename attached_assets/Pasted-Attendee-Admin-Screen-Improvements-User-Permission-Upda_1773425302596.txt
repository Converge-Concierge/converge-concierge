Attendee Admin Screen Improvements + User Permission Updates

We need to improve the Admin → Attendees screen so the team can immediately understand how attendees are categorized, and we also need to update the user permission model to reflect all the new Concierge functionality that has been added.

There are three required changes:

show attendee categorization clearly on the Attendee admin screen

add an attendee category pie chart for the selected event

remove the Bulk Import button because attendee imports are handled through Data Exchange

In addition, we need a broader user permission update so the admin/security model matches the expanded platform.

Primary Goal

Update the Attendees admin page so that it is more useful operationally, and update the permissions model so all major Concierge modules and actions are properly controlled.

1. Add Attendee Category to the Admin Attendee Screen

We now need to clearly show how each attendee is categorized.

The important normalized attendee categories are:

Practitioner

Government / Non-Profit

Solution Provider

Required behavior

Add a new column to the Attendees table:

Category

This should display the normalized attendee category, not just the raw ticket type.

Examples:

Practitioner

Government / Non-Profit

Solution Provider

If needed, store/display the normalized field from the attendee record, such as:

attendee_category

This should make it immediately obvious how each attendee is classified.

2. Keep Original Ticket Type Available (Optional but Recommended)

If the source data still includes the original Eventzilla ticket type, keep it available in the attendee detail/edit view.

Example:

Ticket Type: Practitioner - Team Registration (3 or More People)

Category: Practitioner

This is useful for admin verification and troubleshooting, but the main grid should prioritize the normalized Category field.

3. Add Attendee Category Pie Chart to the Attendees Page

We want a quick visual summary of attendee composition by event.

Required behavior

Add a pie chart near the top of the Attendees page for the currently selected event tab.

Chart title
Attendee Category Breakdown
The chart should show

Practitioner

Government / Non-Profit

Solution Provider

It should also show counts and/or percentages

Example:

Practitioner — 62%

Government / Non-Profit — 14%

Solution Provider — 24%

This should update dynamically based on the selected event tab.

4. Event-Specific Behavior for the Pie Chart

When the admin is viewing:

FRC2026

TLS2026

USBT2026

CUGI2027

the pie chart should reflect only that selected event.

When viewing:

All Events

the chart can either:

aggregate across all events
or

show a combined total for all visible attendee records

That is acceptable, as long as it is clearly labeled.

5. Placement of the Pie Chart

Place the pie chart above the attendee table, ideally near the top summary area.

Recommended layout:

event tabs

search/filter row

attendee category pie chart

attendee table

If needed, the pie chart can sit in a summary card or panel so it feels integrated with the page design.

6. Remove the Bulk Import Button

The Bulk Import button should be removed from the Attendees page.

Reason

Attendee import is handled through Data Exchange, so keeping a Bulk Import button here is redundant and potentially confusing.

Required behavior

Remove the Bulk Import button entirely from the Attendees page header.

Keep:

Add Attendee

Remove:

Bulk Import

Do not remove actual import capability from Data Exchange.

7. Optional Improvement — Add Small Summary Counts

If practical, add small summary metrics above the chart or near the chart:

Practitioners

Government / Non-Profit

Solution Providers

Total Attendees

These counts should match the selected event filter.

This is optional, but useful.

15. Update the Access Control UI

The Access Control / Users area should show the new permissions cleanly.

If the current UI is too limited, update it so admins can manage the expanded permission set without confusion.

At minimum:

new permissions must be visible

they must be assignable

they must persist correctly

16. Acceptance Criteria

This work is complete when:

the Attendees admin table includes a clear Category column

the Attendees page includes an Attendee Category Breakdown pie chart

the chart updates by selected event tab

the Bulk Import button is removed from Attendees

the permissions system is updated to reflect all major new Concierge modules and actions

new matchmaking, sponsor dashboard, deliverables, reporting, and backup features are covered by permissions

17. Success Standard

After this update:

admins can immediately see attendee category composition by event

attendee classification is operationally useful for matchmaking and event planning

the Attendees page is cleaner

the permissions model matches the real scope of the platform

That is the standard this update should meet.
