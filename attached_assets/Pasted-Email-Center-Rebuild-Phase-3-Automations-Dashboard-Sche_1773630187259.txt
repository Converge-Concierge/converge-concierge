Email Center Rebuild — Phase 3
Automations Dashboard & Scheduled Email Engine

We are now moving to Phase 3 of the Email Center rebuild.

Phase 3 focuses on visibility and control of automated system emails.

The Concierge platform already sends some automated emails such as meeting confirmations, reminders, and deliverable notifications. Currently, these are largely invisible or difficult to manage.

This phase introduces an Automations Dashboard that allows Converge to:

see all automated emails in the system

understand what triggers them

turn automations on or off

pause all automations if necessary

monitor automation activity

The objective is control and transparency, not creating dozens of new automation rules.

Phase 3 Goal

Transform the Scheduled Emails section into a true Automation Management System that gives Converge full visibility and control over all system-generated emails.

1. Add a New Tab in Email Center

Add a new tab within Email Center called:

Automations

Email Center structure should now be:

Email Center
   Email Activity
   Templates
   Automations
   Scheduled (existing)

The Automations tab will become the main automation management interface.

The existing Scheduled section may remain temporarily but will eventually be merged.

2. Build the Automations Dashboard

The Automations tab should contain a dashboard listing all automated email workflows.

Each row should represent one automation rule.

3. Automation Table Structure

The Automations dashboard should include a table with the following columns:

Automation Name

Category

Trigger

Audience

Template

Status

Last Run

Emails Sent

Actions

4. Automation Categories

Add logical categories for automations.

Recommended categories:

Meeting

Meeting Confirmation

Meeting Reminder (24 hours)

Meeting Reminder (2 hours)

Info Requests

Info Request Confirmation

Info Request Notification

Sponsor

Sponsor Dashboard Welcome

Deliverable Reminder

Attendee

Scheduling Invitation

Profile Completion Reminder (future)

System

Sponsor Performance Report

Administrative alerts

These categories help admins understand the purpose of each automation.

5. Display Automation Trigger Logic

Each automation should display its trigger condition.

Examples:

Meeting Confirmation
Trigger: Meeting Created
Audience: Attendee + Sponsor
Meeting Reminder – 24 Hours
Trigger: 24 hours before scheduled meeting
Audience: Meeting Participants
Deliverable Reminder
Trigger: Deliverable incomplete after deadline
Audience: Sponsor

The trigger should be clearly visible so admins understand when emails fire.

6. Add Status Controls

Each automation should include a Status toggle.

Statuses:

Enabled
Disabled

Admins should be able to toggle automations on/off without modifying code.

Example:

Meeting Reminder – 24 Hours
Status: Enabled
7. Add Global Automation Control

At the top of the Automations page add system-wide controls:

Pause All Automations
Resume All Automations

This allows Converge to immediately stop all automated messaging if necessary.

Example situations:

event configuration issue

template error

accidental bulk sends

When paused, all automations should be temporarily disabled but not deleted.

8. Add Automation Detail View

Each automation should support a View Details action.

The detail view should show:

Automation Name

Category

Trigger condition

Audience

Template used

Event scope (if applicable)

Last run timestamp

Total emails sent

Status

Optional but useful:

last 5 execution logs

last error message if applicable

This helps with debugging automation behavior.

9. Track Automation Activity

Add basic automation metrics:

Each automation should track:

Last Run

Emails Sent

Failures (if available)

These metrics should appear in the Automations table.

Example:

Deliverable Reminder
Last Run: Mar 15 02:00 AM
Emails Sent: 12
10. Connect Automations to Templates

Each automation must show which template it uses.

Example:

Meeting Reminder – 24 Hours
Template: meeting_reminder_24h

Admins should be able to click the template name to open it in the Templates section.

This makes troubleshooting easier.

11. Link Automations to Email Activity

Emails generated by automations should appear in Email Activity with:

Source: Automation

If possible, also include the automation name.

Example:

Source: Automation – Meeting Reminder 24h

This makes Email Activity easier to understand.

12. Expose Existing Automations First

Before adding new automation types, surface the ones that already exist in Concierge.

Examples likely include:

Meeting confirmation

Meeting reminder (24h)

Meeting reminder (2h)

Info request confirmation

Deliverable reminder

Sponsor dashboard welcome

Scheduling invitation

If additional automations already exist in code, include them as well.

The objective is to make existing behavior visible.

13. Automation Safety Behavior

If an automation is disabled:

no new emails should be triggered by that automation

previously scheduled emails may still send depending on system architecture

Document this behavior clearly in the UI if needed.

14. Preserve System Stability

Automations should not break existing functionality.

If an automation currently runs successfully:

the dashboard should control it

but should not interrupt its logic unless disabled.

Ensure the system fails safely.

15. Add Basic Automation Logging

Automations should record execution events.

Minimum information to log:

automation name

execution timestamp

emails sent

status (success/failure)

These logs can be visible in the detail view or available internally.

16. Ensure Event Context Where Applicable

Some automations apply per event.

Example:

Scheduling Invitation – FRC2026

Where possible, show event context.

If an automation is global across events, display:

Scope: All Events
17. UI Design Goals

The Automations Dashboard should be simple and operational.

Admins should quickly understand:

what emails are automated

when they trigger

whether they are active

how often they run

Avoid technical language where possible.

Use clear labels.

18. Acceptance Criteria

Phase 3 is complete when:

Email Center includes a new Automations tab

all existing automated emails are listed in the Automations dashboard

each automation shows trigger logic and template used

admins can enable or disable individual automations

admins can pause all automations globally

automation activity metrics are visible

automation-generated emails appear in Email Activity with source attribution

19. Success Standard

After Phase 3, Converge should be able to clearly answer:

what automated emails exist

what triggers them

which template they use

whether they are active

how often they run

That is the standard this phase should meet.
