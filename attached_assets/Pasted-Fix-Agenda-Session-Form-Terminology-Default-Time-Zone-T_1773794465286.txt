Interest Topics

Agenda Topics

Central (America/Chicago)

3. Fix TLS2026 Agenda Topic Save Behavior in Admin → Agenda

There is a save bug for TLS2026.

Problem

For TLS2026, when Agenda Topics are selected in the session edit form and the user clicks:

Save Changes

the selected topics do not save.

This must be fixed.

Required behavior

When editing a TLS2026 session and selecting Agenda Topics:

clicking Save Changes must persist those topic selections

reopening the same session should show those topics still selected

recommendation logic should then be able to use those saved mappings

Audit likely causes

Please audit the actual save flow and find the real issue. Likely causes may include:

Possible issue A

The UI selection state is changing visually, but the selected topic ids are not being included in the save payload.

Possible issue B

The save route works for some events but fails for TLS2026 because of event-scoping logic.

Possible issue C

The topic mapping save is only partially wired and the main session save route is not persisting topic selections for TLS2026.

Possible issue D

The TLS2026 Agenda Topics are being loaded correctly, but their ids are not matching the save/update logic.

Possible issue E

The backend save succeeds for the session record but silently fails for the session-topic mapping table.

Find the actual cause and correct it.

Required QA

After the fix, test TLS2026 specifically by:

opening a TLS2026 session

selecting one or more Agenda Topics

clicking Save Changes

reopening the session

verifying those topics remain selected

Do not mark this complete without verifying it persists.
