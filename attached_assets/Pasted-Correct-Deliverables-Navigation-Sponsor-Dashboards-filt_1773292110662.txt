Correct Deliverables Navigation, Sponsor Dashboards filtering, Sponsor Dashboard tab/order/layout, and Rebuild Deliverable-Specific Sponsor Workflows

We need to correct several issues in the current implementation and properly align the system to the intended Deliverables / Sponsor Dashboard structure.

This is a refinement and correction phase. Preserve working logic where possible, but fix the items below.

1. Sponsorship Templates Navigation Fix
Problem

The new Sponsorship Templates menu item was created, but the Sponsorship Templates tab still appears inside Deliverables.

That is not the intended structure.

Required behavior

The Sponsorship Templates menu item should be the only place that handles sponsorship template management.

Therefore:

remove the Sponsorship Templates tab from inside Deliverables

keep the admin menu item:

Configuration → Sponsorship Templates

keep Deliverables focused on:

Sponsor Agreements

Sponsor Fulfillment Queue

Outstanding Sponsor Items

related live operational views

Acceptance

When clicking Deliverables, the user should not see a Sponsorship Templates tab anywhere inside that module.

2. Sponsor Dashboards Event Filtering Fix
Problem

The Sponsor Dashboards page is showing eGain under TLS2026, but eGain is not a TLS sponsor.

This means the event filtering / tab logic is incorrect.

Required behavior

The Sponsor Dashboards list must only show sponsors that are actually assigned to the selected event.

Fix filtering logic

When an event tab is selected:

only include sponsors where that event is one of the sponsor’s assigned events

do not show sponsors from other events

do not infer event assignment loosely from meetings or other activity

Example

If TLS2026 is selected:

show only sponsors assigned to TLS2026

If FRC2026 is selected:

show only sponsors assigned to FRC2026

If All Events is selected:

show all active sponsor dashboards

Acceptance

Selecting TLS2026 should not show eGain unless eGain is explicitly assigned to TLS2026.
