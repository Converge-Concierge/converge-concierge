Create a Concierge Demo Data Blueprint

We are building a separate Concierge demo environment. It needs a consistent, realistic, and sales-friendly demo dataset that can be reseeded repeatedly.

The goal is not just to fill the app with fake data. The goal is to create demo stories that clearly show the value of Concierge across:

sponsor dashboards

attendee scheduling

info requests

deliverables

reporting

admin operations

This blueprint should be used by the demo seed script so the demo is always reset to a polished, presentation-ready state.

Primary Goal

Seed the demo environment with enough realistic sample data to demonstrate:

sponsor onboarding

deliverables tracking

attendee discovery and meeting requests

information requests

sponsor reporting

admin dashboard monitoring

deliverables fulfillment and follow-up

The demo should feel like a real live event environment, but use only safe fictional data.

16. Demo Reset Expectations

The seed dataset should be deterministic enough that after reset:

the same demo stories still work

the same named sponsors/events exist

the same event tabs are available

reports continue to look good

Some randomized variation is fine, but the core demo story should stay stable.

17. Add Internal Labels to Demo Entities Where Helpful

Keep the data realistic, but where helpful include “Demo” in event names or metadata to avoid confusion.

For example:

Demo Fintech Risk & Compliance Forum 2026

Do not make every company sound fake/cartoonish. Keep sponsors and attendee companies plausible.

18. Acceptance Criteria

This demo data blueprint is complete when:

three demo events exist

12 realistic sponsors exist

75–100 attendees exist

sponsor dashboards have meaningful data

deliverables have mixed completion states

meetings and info requests exist

reports look realistic

admin dashboard shows meaningful operational data

multi-event sponsor examples exist

the environment can be reset and return to the same strong demo state

19. Success Standard

After this blueprint is implemented, Concierge demos should consistently feel like a live, high-value sponsor management platform rather than a thin sandbox.

The demo should support strong sales storytelling across:

sponsor onboarding

deliverables

meetings

info requests

admin monitoring

sponsorship reporting

That is the standard this demo blueprint should meet.
