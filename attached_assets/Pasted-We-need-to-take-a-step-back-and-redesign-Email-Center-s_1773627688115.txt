We need to take a step back and redesign Email Center so it becomes the central messaging system for the entire platform, not just a passive log viewer.

Right now Email Center has a useful foundation, but it is too limited. Concierge now has enough functionality that email is becoming a core operational layer for:

attendee engagement

sponsor communications

system notifications

reminders

reporting

future campaigns and automations

Email Center should now evolve into the Operational Communications Hub for Concierge.

1. Primary Goal

Redesign Email Center so it manages:

A. System Emails

Automatic emails generated by system actions.

Examples:

Meeting confirmation

Meeting reminders

Info request confirmation

Sponsor dashboard login

Scheduling invitations

B. Operational Emails

Emails sent manually by Converge admins.

Examples:

Invite attendees to schedule meetings

Remind sponsors to complete deliverables

Send agenda announcements

Send sponsor reports

C. Campaign Emails

Targeted event communications.

Examples:

Invite attendees who have not scheduled meetings

Remind attendees to complete their profile

Notify sponsors of new attendees

Notify attendees of recommended sessions

Email Center should no longer feel like “logs and templates.”
It should feel like the communication command center for Concierge.

10. Template Preview

Each template should support Preview using sample data.

This should render the actual email with:

subject

body

variables filled with sample values

This prevents broken templates and makes editing much safer.

11. Send Test Email

Each template should allow admins to send a test email to themselves.

This should work from:

template row action

template edit view

Do not rely only on a generic Send Test Email button.

Each template should support its own test send.

12. Variables Documentation

Each template should show available variables/tokens.

Example:

Available Variables

{attendee_first_name}
{event_name}
{sponsor_name}
{meeting_time}
{meeting_location}

This should be visible in the edit view and ideally grouped by context.

This will prevent template mistakes and broken sends.

13. Template Version History

Add template versioning.

Admins should be able to:

see prior versions

compare versions if practical

restore a previous version

This is important because admins will eventually break templates accidentally.

28. Scheduled Emails Tab — One-Time and Rule-Based Sends

Scheduled should support both:

One-time scheduled emails

Example:

Rule-based automations

Example:

send reminder 3 days after invitation if still no meetings

send sponsor reminder if deliverables incomplete

send meeting reminders 24h / 2h before

This area should show both clearly.

If implementation must be phased, do these first:

These provide the highest ROI and will make Email Center feel professional and operationally useful.

36. Acceptance Criteria

This rebuild is complete when:

Email Logs is renamed to Email Activity

Email Activity includes delivery/open/click visibility

Email Activity has strong filtering and email detail views

Templates support preview, test email, variable documentation, and version history

Campaigns exists as a new section with audience targeting

Scheduled becomes a real scheduling/automation engine

an Automation Dashboard clearly shows all automated emails

every automation can be turned on or off

global pause/resume for automations exists

Email Settings exists with sender configuration and domain visibility

safety rules are in place for sponsor-protected emails

Email Center feels like the communications hub of Concierge, not just a log viewer

37. Success Standard

After this rebuild, Email Center should function as the operational communications system for Concierge, giving Converge full visibility and control over system emails, admin emails, campaigns, reminders, and automations across the platform.
