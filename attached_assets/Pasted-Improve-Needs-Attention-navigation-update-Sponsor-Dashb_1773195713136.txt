1. Admin Dashboard — “Needs Attention” → “View Sponsors” must open the relevant filtered sponsor results
Problem

On the Admin Dashboard, clicking:

Needs Attention → View Sponsors

currently just opens the general Sponsors admin page.

It does not identify which sponsor(s) have no meetings, which makes the alert much less useful.

Required behavior

When the user clicks:

View Sponsors

from the dashboard alert:

1 sponsor with no meetings

the system should open the Sponsors page already filtered to the relevant sponsor set.

The user should immediately be able to tell:

which sponsor(s) triggered the alert

why they are in the result set

Recommended implementation

Use one of these approaches:

Preferred approach

Route to Sponsors page with a query/filter state such as:

/sponsors?attention=no-meetings

Then the Sponsors page should:

apply the filter automatically

show only sponsors with zero meetings across the relevant scope

show a filter pill / label indicating the active alert filter

Example helper text at top of table:

Showing sponsors with no meetings

Acceptable alternative

If server-side filtering is harder, route to the Sponsors page and apply a client-side filter based on the alert condition.

But the page must still clearly show that the sponsor list is filtered.

Required visual indication

Once on the Sponsors page, show a visible indicator that a special filter is active.

Examples:

filter chip:

No Meetings

banner:

Showing sponsors with no meetings

badge near search/filter controls

Do not silently route to the same full sponsor list.

Required filter logic

At minimum, the filter should identify sponsors who currently have:

zero meetings across all events
or

zero meetings in the current dashboard scope if your dashboard is event-scoped

Use the same logic as the dashboard alert so the counts match.

Acceptance criteria

clicking View Sponsors from the alert opens a filtered sponsor list

the relevant sponsor(s) are visible immediately

the Sponsors page clearly indicates the active “no meetings” filter

the result set matches the dashboard alert count

3. Sponsor Dashboard — “Your Meetings” statuses should be editable / reversible
Problem

On the Sponsor Dashboard, in Your Meetings, the meeting status behaves too much like a fixed label.

We need it to work more like Information Requests, where the status can be changed directly.

Required behavior

Inside the Your Meetings section on the Sponsor Dashboard, allow authorized sponsor users to update meeting status inline.

This should be similar to how Information Request status editing works.

Recommended editable meeting statuses

Use the actual meeting statuses supported by Concierge.

Recommended list:

Scheduled

Confirmed

Completed

Cancelled

No-Show

Pending

Use only statuses that already fit the actual meeting model in Concierge.

If “Pending” is not valid for confirmed meetings in your current model, exclude it.

Recommended default list for sponsor dashboard

If aligned with current system, use:

Scheduled

Confirmed

Completed

Cancelled

No-Show

UI behavior

Replace fixed status display with one of these:

Preferred

Inline dropdown / select in the Status column

Acceptable

Click status badge → open status menu

The interaction should be fast and clean.

Permission requirements

Meeting status editing should respect sponsor access levels.

Owner

Can edit meeting status

Editor

Can edit meeting status

Viewer

Cannot edit meeting status

For viewers:

show status as read-only

do not show edit controls

Reversible / editable requirement

The status should be changeable, not locked after first update.

Example:

Scheduled → Completed

Completed → Confirmed

Confirmed → Cancelled

This means status updates must be reversible if a sponsor or admin needs to correct them.

Audit / logging requirement

If audit logging exists, log:

old status

new status

updated by

timestamp

If not already available, at least update cleanly in DB and preserve updated_at.

Acceptance criteria

sponsor owners and editors can update meeting status from the Sponsor Dashboard

viewers cannot edit status

meeting status can be changed more than once

updated status saves correctly and persists on reload
