Optimize Sponsor Dashboard to Be More Impressive, More Useful, and Fix Notifications

Update the current Converge Concierge Sponsor Dashboard so it feels more like a premium sponsor ROI and lead-management portal, not just a meeting summary page.

The current dashboard already includes:

sponsor identity header

KPI cards

meeting status chart

meeting type chart

notifications section

meetings table

lead contacts section

information requests section

sponsor performance report

export buttons

This release should improve the dashboard in two ways:

Fix broken notification behavior

Upgrade the dashboard UX and utility so sponsors immediately understand their event ROI and next actions

Primary Goals
Goal 1

Fix the “2 new” notification pill in the top header so it performs a useful action when clicked.

Goal 2

Improve the Sponsor Dashboard so it better helps sponsors:

understand current results

prepare for upcoming meetings

manage follow-up

see lead value

feel that Concierge is a serious business tool

Scope Rules
Do build

functional notification pill behavior

stronger top-level KPI framing

Next Meeting widget

more actionable Notifications behavior

better lead and follow-up visibility

clearer sponsor performance reporting

cleaner information hierarchy

Do not build yet

recommended attendees engine

AI scoring

CRM integrations

complex new backend scoring models unless trivial

This phase is a dashboard optimization release, not a new product module build.

1. Fix the Top “2 New” Notification Button

The purple notification pill in the top header currently does nothing when clicked.

Required behavior

When the user clicks the notification pill:

scroll to the Notifications section on the dashboard

expand the Notifications panel if it is collapsed

visually highlight unread notifications

optionally set focus on the panel heading

If smooth scrolling is already used in the app, use it here.

Better behavior if easy to support

If straightforward, make the top notification pill open a small dropdown preview containing the latest unread notifications, with a link such as:

View all notifications

If this is more effort than desired, use the scroll-to-panel behavior for now.

Notification interaction improvements

Each notification row should become clickable.

Clicking a notification should open the most relevant related destination, for example:

related meeting

related information request

related sponsor action area

At minimum, clicking a notification should:

mark it read

visually remove the unread dot or unread emphasis

3. Add a “Next Meeting” Widget

Add a new high-visibility card near the top of the dashboard.

This should be one of the most prominent blocks on the page.

Purpose

Show the sponsor exactly what is next on their schedule.

Card content

attendee full name

title

company

date

time

meeting type

location or platform

meeting status

Actions

Include one or more of these actions if already supported:

View Meeting

Add to Calendar

Open Zoom / platform link if online

Mark Completed when applicable

Empty state

If the sponsor has no upcoming meetings, show:

No upcoming meetings yet

Optional helper text:
New meetings and requests will appear here when scheduled.

4. Update KPI Cards to Reflect Sponsor ROI

The current metrics are useful but should be labeled and ordered more clearly around sponsor outcomes.

Recommended KPI set

Use these cards in this order:

Meetings Scheduled

Meetings Completed

Companies Engaged

Leads Captured

Information Requests

If there is strong reason to keep “Pending Online,” move it to a secondary area instead of the main row.

Metric definitions

Meetings Scheduled = total current meetings

Meetings Completed = completed meetings only

Companies Engaged = distinct companies represented by attendees met or scheduled

Leads Captured = count of unique lead contacts or engaged attendee records

Information Requests = total information requests submitted to this sponsor

Visual requirement

Cards should show:

metric number

label

optional small helper text or icon

clean, premium styling consistent with the rest of the platform

If trend deltas are not available, do not fake them.

5. Make Notifications More Actionable

The Notifications section currently acts more like a passive feed.

Convert it into an operational activity center.

Required improvements

Each notification should display:

event type

attendee name

company

date/time

notification date

read/unread state

Notification types may include

Request confirmed

Online request received

Onsite meeting booked

Information request received

Meeting cancelled

Meeting updated

Notification actions

Where relevant, include inline action links or buttons such as:

View Meeting

View Request

Mark Read

If inline actions make the UI too crowded, make the entire row clickable and add a kebab or secondary action.

Mark all read

Keep Mark all read, but ensure it actually updates notification state immediately in the UI and in persistent storage.
