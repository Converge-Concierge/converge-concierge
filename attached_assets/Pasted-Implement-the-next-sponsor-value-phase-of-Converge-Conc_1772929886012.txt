Current goal:
Add 3 major sponsor-value features:

2. Calendar integration

==================================================
1. SPONSOR NOTIFICATIONS
==================================================

Add sponsor-facing notifications for meeting activity.

Trigger notifications for:
- new onsite meeting booked
- new online meeting request submitted
- meeting cancelled
- online meeting request confirmed
- online meeting request declined

At minimum, notifications must be visible inside the sponsor access dashboard.
If email sending is already available in the project, also prepare email notification hooks, but do not block this feature if email delivery is not yet configured.

Add a notifications area to sponsor access pages showing:
- notification type
- attendee name
- company
- event
- date/time
- read/unread state
- timestamp

Suggested notification types:
- onsite_booked
- online_request_submitted
- meeting_cancelled
- request_confirmed
- request_declined

Do not expose one sponsor’s notifications to another sponsor.

==================================================
2. CALENDAR INTEGRATION
==================================================

For confirmed meetings, add calendar support.

Applies to:
- onsite meetings with confirmed/scheduled status
- online meetings when status becomes Confirmed

Add options:
- Download ICS
- Add to Google Calendar (if easy to generate)
- Add to Outlook (if easy to generate)

At minimum, implement ICS generation.

ICS should include:
- meeting title
- sponsor name
- attendee name
- date
- start time
- end time (default 30 minutes)
- location for onsite meetings
- platform / online meeting note for online meetings
- event code and event name in description

Where to show calendar actions:
- public confirmation page for onsite meetings
- public confirmation page for online meeting requests after confirmation if available
- sponsor dashboard meeting detail if confirmed
- admin meeting detail if confirmed

Do not break existing confirmation screens.

3. SPONSOR PROFILE PAGES
==================================================

Create public sponsor profile pages linked from sponsor cards.

When an attendee clicks a sponsor card or “Learn More” action, show a sponsor profile page before booking.

Route example:
- /event/:eventCode/sponsor/:sponsorId

Sponsor profile should display:
- sponsor logo
- sponsor name
- sponsorship level
- short description
- website (optional)
- LinkedIn URL (optional)
- product / solution summary (optional)
- event they are assigned to
- buttons:
  - Schedule Onsite Meeting
  - Request Online Meeting (only if sponsor allows online meetings)

Add these editable sponsor fields in admin:
- shortDescription
- websiteUrl
- linkedinUrl
- solutionsSummary

Do not require all new fields.
Keep them optional.

If a sponsor does not have extra profile content, the page should still render cleanly using name, logo, level, and booking actions.
