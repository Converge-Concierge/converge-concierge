Make “Forgot Password?” Fully Functional on the Admin Login Panel

Implement a working Forgot Password flow for the Admin Login panel in Concierge.

Right now the Forgot password? link appears on the admin login screen, but it does not operate as a complete recovery workflow.

We need a standard, secure password reset flow for admin users.

Primary Goal

When an admin clicks Forgot password?, they should be able to:

enter their work email

receive a password reset email

click a secure reset link

create a new password

return to login and sign in successfully

This should be secure, token-based, and integrated with the existing Brevo email setup.

Scope Rules
Do build

forgot password request form

secure reset token generation

password reset email

reset password page

token validation

password update logic

success and failure messaging

Do not build

SMS reset

security questions

magic-link login

admin approval workflow

This should be a standard email-based password reset flow.

1. Make the “Forgot password?” Link Clickable

On the admin login screen, the Forgot password? text should route to a dedicated password reset request page or modal.

Preferred route

Use a dedicated route such as:

/admin/forgot-password

This is cleaner and easier to maintain than a modal.

2. Build Forgot Password Request Page

Create a page:

Forgot Password

Helper text:

Enter your work email and we’ll send you a link to reset your password.

Form fields

Work Email

Buttons

Send Reset Link

Back to Sign In

Keep the form simple and aligned with the existing admin login styling.

3. Lookup Admin User by Email

When the forgot password form is submitted:

normalize the email address

look up an admin user account by that email

if found, generate a password reset token

send a password reset email

Important:
Do not reveal whether the email exists in the system.

Always show a neutral success message like:

If an account exists for that email, a password reset link has been sent.

This prevents user enumeration.

4. Create Password Reset Token Logic

When a valid forgot password request is made:

generate a secure random token

hash it before storing if possible

store token metadata in the database

Suggested reset token fields:

id
user_id
token_hash
expires_at
used_at nullable
created_at

You may use a dedicated table such as:

password_reset_tokens
Token rules

token must expire

token should be single use

token should be invalid once used

token should be invalid after expiration

Recommended expiration
1 hour

or

2 hours

1 hour is preferred.

5. Send Password Reset Email via Brevo

Use the existing Brevo email integration.

Create a new email template for password reset.

Subject

Reset your Concierge password

Body should include

greeting

short explanation

reset button

expiration notice

fallback plain link

Example helper text:

We received a request to reset your Concierge admin password. Click the button below to create a new password. This link expires in 1 hour.

CTA button

Reset Password

The reset link should point to a route like:

/admin/reset-password?token=...
6. Build Reset Password Page

Create a page:

/admin/reset-password

This page should read the token from the URL.

Page behavior

validate token

if valid, show password reset form

if invalid or expired, show clear error state

Reset form fields

New Password

Confirm New Password

Button

Reset Password

7. Password Validation Rules

Enforce a reasonable password policy.

Recommended minimum:

at least 8 characters

Recommended stronger policy:

at least 8 characters

one uppercase letter

one lowercase letter

one number

If the app already has a password policy, reuse it.

Show inline validation and a clear mismatch message for confirm password.

8. Complete Password Reset

When the user submits a valid new password:

validate token again

validate password rules

hash the new password securely

update the admin user password

mark the reset token as used

invalidate any other active reset tokens for that user if practical

redirect to login with success message

Success message

Your password has been reset. You can now sign in.

9. Invalid / Expired Token Handling

If token is:

missing

invalid

already used

expired

show a clear message like:

This password reset link is invalid or has expired. Please request a new reset link.

Include a button:

Request New Reset Link

Do not show raw system errors.

10. Security Requirements
Required

use cryptographically secure token generation

do not store raw passwords

hash passwords with the same secure hashing method already used by the app

do not expose whether a user exists during forgot password request

token must expire

token must be single-use

Recommended

log password reset requests

rate-limit forgot password requests

rate-limit reset attempts

11. Add Email Template for Password Reset

Create a new email template in the same structure used for Concierge emails.

Suggested template key:

password_reset

Brand it consistently with:

Converge Concierge

clean layout

simple CTA

footer note

12. Add Email Logging

Log the password reset email in the existing email_logs system.

Use email type such as:

password_reset

This helps with troubleshooting.

13. Optional Audit Logging

If audit logging already exists, log events like:

password reset requested

password reset completed

Do not log raw passwords or raw tokens.

14. Add Rate Limiting / Abuse Protection

Add lightweight protection for forgot password requests.

At minimum:

throttle repeated reset requests for the same email

throttle repeated requests from the same IP if practical

Example policy:

no more than 3 requests per 15 minutes per email

Keep it simple but present.

15. UI / UX Requirements

Match the styling of the existing login page.

Forgot Password request page

same card style

same typography

same button style

Reset Password page

same structure and polish

clear status and validation messaging

The flow should feel like part of the same admin authentication system.

16. Recommended Routes

Use routes like:

/admin/login
/admin/forgot-password
/admin/reset-password

If your current auth routes use a different namespace, stay consistent, but keep the flow clean.

17. Recommended Implementation Order

Implement in this order:

Step 1

Create forgot password request page and route

Step 2

Create password reset token storage/model

Step 3

Create password reset email template and sending logic

Step 4

Create reset password page and token validation

Step 5

Implement password update and token invalidation

Step 6

Add email logging, polish messaging, and test edge cases

18. Acceptance Criteria

This task is complete when:

clicking Forgot password? opens a real recovery flow

admin can submit work email for reset

system sends password reset email via Brevo

reset link opens a valid reset password page

token validation works correctly

expired/invalid links are handled cleanly

new password is saved securely

admin can log in with the new password

system does not reveal whether an email exists at request time

19. Success Standard

An admin who forgets their password should be able to recover access securely and smoothly without manual intervention.

That is the standard this feature should meet.
