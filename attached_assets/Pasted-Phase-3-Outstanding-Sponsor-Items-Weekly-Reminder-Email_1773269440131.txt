Phase 3 — Outstanding Sponsor Items + Weekly Reminder Emails + Admin Follow-Up Workflow

Phase 2 is being implemented. The next step is to operationalize Agreement Deliverables so Julie and the team can easily manage outstanding sponsor inputs and the system can automatically remind sponsors when action is needed.

This phase should make Agreement Deliverables usable as an ongoing sponsor follow-up system, not just a dashboard display.

Primary Goal

Create an admin workflow where the team can:

see all outstanding sponsor-responsible items across sponsors and events

quickly identify which sponsors still owe inputs

send manual reminders when needed

automatically send a weekly reminder email if sponsor-owned deliverables remain incomplete

This phase should improve sponsor activation and reduce manual back-and-forth.

Scope for Phase 3
Build now

Agreement Deliverables → Outstanding Sponsor Items admin screen

filters and actions for follow-up

manual reminder send from admin

weekly reminder email job

reminder eligibility logic

sponsor reminder email template

reminder logging

dashboard/alert tie-ins where appropriate

Do not build yet

Cloudflare R2 uploads

deliverable file attachments

Zoho Projects sync

sponsor-facing activity timeline

advanced analytics / trend charts

SMS reminders

2. Define Reminder Eligibility Logic

Only certain deliverables should trigger reminder emails.

The system should send reminders only for sponsor-responsible items that still require sponsor action.

2A. Reminder-eligible deliverables

A deliverable is eligible for reminder if:

reminder_eligible = true

sponsor_visible = true

owner_type = Sponsor
or

owner_type = Shared AND currently blocked on sponsor input

And the status is one of:

Awaiting Sponsor Input

Not Started

Needed

Issue Identified

Blocked (only if blocked because of sponsor input)

2B. Exclude from reminders if:

status = Delivered

status = Approved

status = Scheduled (if no sponsor action needed)

status = Available After Event

owner_type = Converge only

sponsor has no active sponsor users

2C. COI logic

For Certificate of Insurance, include in reminders only when status is:

Needed

Issue Identified

Exclude when:

Received

Under Review

Approved

4. Build Weekly Reminder Email Job

Implement a scheduled process that runs once per week and sends reminder emails to sponsors with outstanding sponsor-owned deliverables.

4A. Frequency

Run weekly on a fixed schedule.

Recommended:

every Tuesday morning
or

every Wednesday morning

Pick one and keep it consistent.

4B. Recipient logic

Initially send reminders to:

the primary sponsor Owner only

Later you may optionally include Editors, but do not do that now.

4C. Weekly reminder grouping

Do not send one email per deliverable.

Instead:

group all outstanding reminder-eligible items for a sponsor + event into one weekly email

This is cleaner and more professional.

Example:

Winnow / FRC2026 gets one reminder email listing all outstanding items

4D. Do not send if:

sponsor has no outstanding reminder-eligible items

sponsor has no active owner user

reminder was already sent in the same weekly cycle if you track cycles

sponsorship is inactive / archived if such a concept exists

Use your actual route structure.

6. Build Manual Reminder Send

On the admin side, allow manual sending of reminders.

This should work from:

Outstanding Sponsor Items list

Sponsor Agreement detail screen

6A. Manual send behavior

When admin clicks Send Reminder:

gather all outstanding reminder-eligible items for that sponsor/event

send one grouped reminder email

log it in agreement_deliverable_reminders

show success/failure message

Do not send just for the selected row unless that is the only eligible item.
The better sponsor experience is one grouped reminder email.

6B. Confirmation text

Use a confirmation modal:

Send a reminder to [Sponsor Name] for all outstanding sponsorship items for [Event Name]?

Buttons:

Cancel

Send Reminder

14. Keep File Support Out of Scope

Some reminder-eligible items will eventually involve uploads (logo, headshot, etc.), but in this phase do not implement actual file upload.

The reminder system should work purely off:

status

sponsor-editable text/structured inputs

sponsor-visible deliverables

This keeps the phase focused.

After Phase 3, Julie and the team should be able to:

quickly see what sponsors still owe
